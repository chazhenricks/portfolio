Domain Name Registrars and Hosting Servers

I bought the domain name from Namecheap and use a Digital Ocean droplet to host it. I needed to make sure I had logins to both places to I could make any updates.

Namecheap is the Domain Registrar. They are the broker that you purchase a domain name from and are the intermediary that tells the big Dot Com Database in the sky that "chazhenricks.com" maps to a server address somewhere. Because most of the access to the Internet is controlled by three kids in a trenchcoat - we need an intermediary to act on our behalf to tell The Internet that a new website exists. In reality, The Internet is just a big Database and we need to pay someone to get a new record added to it.

The more important thing I needed to be able to do is ssh into the hosting server to make some updates. The hosting server is some other computer that is accessable to the intrnet via an IP address. When you "ssh" into a server, you are connecting to it remotely from your computer. So I can access the terminal from that computer via the terminal on my computer. The way we can do this without having to constantly type in passwords is by creating an "ssh key" - which is a public/private pair of Computer Science Wizardry. You can think of it like two sides of a BFF necklace. You have one side that no one is allowed to see. You give the other side to this remote server. Then you tell the server "dont let anyone in here if they dont have my side of the BFF necklace." This effectively makes it very hard for someone thats not you to login to the server.
